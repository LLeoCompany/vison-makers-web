[Section 3] Core Logic: 법률 RAG 작동 원리 (Visualizer)
UI: 사용자가 질문을 입력하면 데이터베이스에서 관련 조항(Chunks)을 찾아 답변을 완성하는 과정을 보여주는 인터랙티브 애니메이션.

단계 설명:

Query: 변호사의 자연어 질문 입력.

Retrieval: 내부 판례/기록 데이터베이스에서 유사 문장 정밀 검색.

Generation: 검색된 근거를 바탕으로 논리적인 법률 검토 의견서 초안 작성.

[Section 4] Use Cases: 구체적 도입 시나리오

Case 2. 계약서 리스크 검토: "수백 페이지의 계약서 중 사내 표준 가이드라인에 위배되는 독소 조항을 즉시 탐지합니다."

Case 3. 법률 상담 자동화: "반복적인 단순 법률 질의에 대해 정확한 법적 근거를 바탕으로 초안을 작성합니다."

[Section 5] Security: 철저한 데이터 주권

[Section 6] Conversion: 맞춤형 상담 신청 (Widget)
UI: 우측 하단 고정 또는 하단 섹션 전면 배치.
