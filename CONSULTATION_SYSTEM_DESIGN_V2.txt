시스템 개요
===============================================================================

설계 목표 (검토 결과 반영)
- 마케팅 이론 기반 과학적 전환율 최적화 시스템 구축
- 모바일 우선(Mobile-First) UX 설계로 접근성 극대화
- 단계적 구현(Progressive Implementation) 전략으로 리스크 최소화
- 데이터 기반 의사결정(Data-Driven) 지원 체계 구축

핵심 개선사항
1. 모바일 최적화: 데스크톱 5단계 vs 모바일 3단계 적응형 플로우
2. 하이브리드 모드: 가이드 + 프리 입력 결합 옵션 제공
3. 중도 이탈 방지: 자동 저장 및 스마트 리마인더 시스템
4. 접근성 강화: WCAG 2.1 AA 기준 준수
5. 실시간 분석: 단계별 이탈률 추적 및 A/B 테스트 지원

비즈니스 임팩트
- 전환율 향상: 30-40% (모바일 최적화 효과)
- 상담 품질: 50-60% (구조화된 데이터 수집)
- 업무 효율성: 40-50% (사전 분류 및 자동화)
- 고객 만족도: 25-35% (개인화된 경험 제공)

상태 관리 시스템 (강화)
- Zustand + React Query: 복잡한 상태 관리 및 서버 동기화
- IndexedDB: 오프라인 지원 및 대용량 데이터 로컬 저장
- Session Recovery: 브라우저 종료 시에도 진행 상태 복원
- Real-time Backup: 30초마다 자동 저장

마케팅 이론 적용 설계
===============================================================================

1. AIDA 모델 기반 단계별 설계

Attention (주의 집중)
- 트랙 선택 페이지: 시각적 대비와 명확한 가치 제안
- "3분 vs 1분" 소요 시간 명시로 선택 동기 부여
- 진행률 표시로 명확한 로드맵 제시

Interest (관심 유발)
- 개인화된 질문: "귀하의 비즈니스에 딱 맞는" 표현 사용
- 실시간 예상 견적: 선택에 따른 대략적 비용 범위 표시
- 성공 사례 미니 케이스: 각 단계별 관련 사례 2-3줄 소개

Desire (욕구 구체화)
- 맞춤형 솔루션 프리뷰: 선택 사항 기반 서비스 개요 생성
- 경쟁 우위 요소: "90% 실패하는 외주, 우리는 다릅니다" 메시지
- 긴급감 조성: "이달 한정 할인" 또는 "빠른 시작 혜택" 암시

Action (행동 유도)
- 마이크로 CTA: 각 단계별 "다음으로" 버튼 최적화
- 손실 회피: "여기까지 진행하셨는데 포기하시겠어요?" 알림
- 사회적 증명: "1,000+ 기업이 선택" 신뢰 지표

2. 행동경제학 원리 적용

손실 회피 (Loss Aversion) 메커니즘
- 진행률 85% 시점: "거의 완료! 5% 할인 혜택까지 바로 앞"
- 이탈 감지: "잠깐! 지금 나가시면 지금까지 입력한 정보가 사라져요"
- 재방문 유도: "저장된 상담 신청서가 있어요. 이어서 진행하시겠어요?"

사회적 증명 (Social Proof)
- "실시간: 지금 3명이 상담 신청 중"
- "이번 주 47건의 상담이 성사되었어요"
- "비슷한 업종 95%가 가이드 상담 선택"

접근성 강화 (WCAG 2.1 AA)
===============================================================================

시각적 접근성
- 색상 대비: 최소 4.5:1 비율 유지
- 색상 의존성 제거: 아이콘+텍스트 병행 표기
- 폰트 크기: 최소 16px, 모바일 18px
- 포커스 표시: 명확한 outline과 배경색 변화

청각적 접근성
- 자막 지원: 소개 영상에 자막 제공
- 음성 피드백: 단계 완료 시 선택적 음성 안내
- 텍스트 대체: 모든 오디오 콘텐츠의 텍스트 버전

운동 기능 접근성
- 키보드 내비게이션: Tab/Enter/Space로 완전 조작
- 터치 타겟: 최소 44px×44px 크기
- 드래그 대체: 모든 제스처의 버튼 대체 방안

인지적 접근성
- 간단한 언어: 전문 용어 최소화 및 설명 제공
- 일관된 레이아웃: 동일한 위치의 버튼과 메뉴
- 진행 상황: 항상 현재 위치와 남은 단계 표시
- 오류 방지: 실시간 유효성 검사 및 명확한 오류 메시지

A/B 테스트 프레임워크
테스트 항목:
1. 트랙 선택 페이지 레이아웃 (3가지 버전)
2. 예산 선택 옵션 (4단계 vs 5단계)
3. 진행률 표시 방식 (바 vs 스텝 vs 퍼센트)
4. CTA 버튼 문구 ("다음으로" vs "계속하기" vs "진행")

사용자 행동 분석
- 히트맵: 각 페이지별 클릭 패턴
- 체류 시간: 단계별 평균 체류 및 이탈 시간
- 스크롤 깊이: 모바일에서 스크롤 행동 패턴
- 재방문 패턴: 미완료 사용자의 재방문 주기

  // 기본 정보
  sessionId: string;
  userId?: string;  // 등록 사용자일 경우
  trackType: 'guided-desktop' | 'guided-mobile' | 'hybrid' | 'free';
  deviceInfo: DeviceInfo;

  // 진행 상태
  progress: {
    currentStep: number;
    totalSteps: number;
    completionRate: number;
    timeSpent: number;  // 초 단위
    lastActiveAt: Date;
  };

  // 수집된 데이터
  consultationData: GuidedData | HybridData | FreeData;

  // 분석 데이터
  analytics: {
    entrySource: string;  // 유입 경로
    browserInfo: BrowserInfo;
    sessionEvents: UserEvent[];  // 클릭, 스크롤 등
    exitPoint?: string;
    completedAt?: Date;
  };

  // 개인화
  preferences: {
    savedForLater: boolean;
    emailReminders: boolean;
    contactTimePreference?: string;
  };
}

interface UserEvent {
  timestamp: Date;
  eventType: 'click' | 'scroll' | 'focus' | 'input' | 'exit_intent';
  element: string;
  value?: string;
  metadata?: Record<string, any>;
}

실시간 알림 자동화
- 고품질 상담 (완성도 80%+): 즉시 알림 + SMS
- 중급 상담 (60-80%): 30분 후 요약 알림
- 기본 상담 (<60%): 시간대별 배치 알림
- VIP 고객 감지: 예산 1500만원+ 즉시 특별 알림

구현 로드맵 (리스크 최소화)
===============================================================================

Phase 0: 준비 단계 (1주)
- 현재 상담 신청 데이터 백업
- A/B 테스트 도구 설치 (Google Optimize)
- 분석 대시보드 기본 설정
- 개발 환경 구축

Phase 1: MVP 구현 (2주)
🎯 목표: 모바일 3단계 + 기본 프리 트랙
- 디바이스 감지 및 라우팅
- 모바일 최적화 가이드 트랙 (3단계)
- 개선된 프리 트랙
- 기본 자동 저장 기능

Phase 2: 고도화 (2주)
🎯 목표: 하이브리드 + 이탈 방지
- 하이브리드 트랙 구현
- 스마트 리마인더 시스템
- 접근성 개선 (WCAG 기준)
- 실시간 분석 연동

Phase 3: 데스크톱 확장 (2주)
🎯 목표: 데스크톱 5단계 + 고급 기능
- 데스크톱용 상세 가이드 트랙
- 고급 A/B 테스트 구현
- 개인화 추천 시스템
- CRM 시스템 연동

Phase 4: 최적화 (1주)
🎯 목표: 성능 + 사용자 경험 개선
- 성능 최적화 및 캐싱
- 사용자 피드백 반영
- 버그 수정 및 안정화
- 모니터링 대시보드 완성

출시 전략
- Soft Launch: 기존 트래픽의 20%만 신규 시스템으로
- 점진적 확대: 주차별 20%씩 트래픽 증가
- 성과 모니터링: 주요 지표 실시간 추적
- 롤백 준비: 언제든 기존 시스템으로 복원 가능

성공 지표 및 KPI
===============================================================================

Primary KPI (1개월 후)
- 전체 상담 완료율: 기존 대비 30% 향상
- 모바일 완료율: 데스크톱 대비 85% 수준 달성
- 상담 품질 점수: 평균 75점 이상 (100점 만점)

Secondary KPI (3개월 후)
- 영업팀 효율성: 초기 상담 시간 40% 단축
- 고객 만족도: 상담 만족도 4.5/5 이상
- 재방문 상담률: 미완료 사용자 재방문 15% 이상

Business Impact (6개월 후)
- 매출 기여도: 신규 상담 시스템으로 인한 매출 증가 측정
- 고객 생애 가치: 체계적 상담을 통한 고객 LTV 향상
- 브랜드 인지도: "쉽고 정확한 상담" 브랜드 이미지 구축
