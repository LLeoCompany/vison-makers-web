시스템 개요
===============================================================================

개선 목표
- 기존 단일 폼 방식에서 투트랙(Dual-Track) 상담 시스템으로 전환
- MiceHub 패턴 기반 구조화된 선택 프로세스 도입
- 클라이언트 유형별 맞춤형 상담 경로 제공

투트랙 구조
1. 가이드 트랙 (Guided Track): 체계적 단계별 선택
2. 프리 트랙 (Free Track): 자유 양식 요구사항 입력

비즈니스 가치
- 상담 품질 향상: 구체적 요구사항 파악
- 전환율 개선: 단계별 진행으로 이탈률 감소
- 업무 효율성: 사전 분류된 상담 데이터

기술 아키텍처
===============================================================================

Frontend 구조 변경
- 기존: Single Page Form (/sub)
- 신규: Multi-Step Consultation Flow

라우팅 구조
/consultation/
├── track-selection      # 트랙 선택 (가이드 vs 프리)
├── guided/             # 가이드 트랙
│   ├── step1-service   # 서비스 유형 선택
│   ├── step2-tech      # 기술 스택 선택
│   ├── step3-timeline  # 일정 및 예산
│   ├── step4-details   # 세부 요구사항
│   └── step5-contact   # 연락처 정보
├── free/              # 프리 트랙
│   ├── requirements   # 자유 양식 요구서
│   └── contact       # 연락처 정보
└── confirmation      # 최종 확인 및 제출

상태 관리
- Context API 또는 Zustand 사용
- 단계별 데이터 상태 관리
- 브라우저 새로고침 시 상태 복원

데이터 구조 설계
===============================================================================

Guided Track 데이터 스키마
interface GuidedConsultationData {
  trackType: 'guided';

  // Step 1: Service Type
  serviceType: {
    category: 'website' | 'webapp' | 'system' | 'design-dev';
    subType: string;
  };

  // Step 2: Tech Stack
  techPreferences: {
    frontend: string[];
    backend: string[];
    specialRequirements: string[];
  };

  // Step 3: Timeline & Budget
  timeline: {
    urgency: 'urgent' | 'normal' | 'flexible' | 'tbd';
    preferredDuration: string;
  };
  budget: {
    range: 'under-300' | '300-800' | '800-2000' | 'over-2000' | 'tbd';
  };
  additionalServices: string[];

  // Step 4: Details
  coreFeatures: string[];
  freeformRequirements: string;

  // Step 5: Contact
  contact: ContactInfo;

  // Meta
  createdAt: Date;
  progressStep: number;
}

Free Track 데이터 스키마
interface FreeConsultationData {
  trackType: 'free';

  // Requirements
  requirements: {
    description: string;
    attachments: FileAttachment[];
  };

  // Contact
  contact: ContactInfo;

  // Meta
  createdAt: Date;
}

Common Contact Interface
interface ContactInfo {
  name: string;
  phone: string;
  email: string;
  company?: string;
  preferredContactMethod: 'phone' | 'email' | 'kakao' | 'meeting';
  preferredContactTime: 'morning' | 'afternoon' | 'evening' | 'weekend';
}

UI 컴포넌트 설계
===============================================================================

공통 컴포넌트
- ConsultationLayout: 진행 상태 표시, 네비게이션
- StepIndicator: 현재 단계 및 진행률 표시
- SelectionCard: 선택 옵션 카드 UI
- ProgressBar: 단계별 진행률
- BackButton: 이전 단계 돌아가기
- NextButton: 다음 단계 진행

가이드 트랙 컴포넌트
- ServiceSelector: 서비스 유형 선택 그리드
- TechStackSelector: 기술 스택 체크박스 그룹
- TimelineBudgetSelector: 라디오 버튼 + 체크박스
- FeatureSelector: 기능 선택 체크리스트
- ContactForm: 연락처 입력 폼

프리 트랙 컴포넌트
- RequirementsEditor: 리치 텍스트 에디터
- FileUploader: 드래그 앤 드롭 파일 업로드
- TemplateHelper: 작성 가이드 템플릿

개발 구현 계획
===============================================================================

Phase 1: 기반 구조 (2주)
- 라우팅 시스템 구축
- 상태 관리 시스템 구현
- 기본 레이아웃 및 공통 컴포넌트
- 트랙 선택 페이지 구현

Phase 2: 가이드 트랙 (3주)
- Step 1-5 페이지 구현
- 단계별 검증 로직
- 데이터 수집 및 저장
- 진행률 표시 시스템

Phase 3: 프리 트랙 (1주)
- 요구사항 에디터 구현
- 파일 업로드 시스템
- 작성 가이드 템플릿

Phase 4: 통합 및 최적화 (1주)
- 두 트랙 통합 테스트
- Slack 알림 시스템 개선
- 사용자 경험 개선
- 성능 최적화

Phase 5: 배포 및 모니터링 (1주)
- 프로덕션 배포
- 사용자 행동 분석 도구
- A/B 테스트 준비
- 모니터링 대시보드

기대 효과 및 성공 지표
===============================================================================

비즈니스 지표
- 상담 신청 완료율: 기존 대비 30% 향상
- 상담 품질 점수: 구체적 정보 제공률 50% 향상
- 영업팀 효율성: 사전 분류로 초기 상담 시간 40% 단축

사용자 경험 지표
- 이탈률 감소: 단계별 진행으로 중도 이탈 20% 감소
- 완성도 향상: 가이드 트랙 사용자의 상담 만족도 25% 상승
- 재방문률: 미완성 상담 재방문율 15% 향상

기술적 지표
- 페이지 로딩 속도: 각 단계 2초 이내 로딩
- 모바일 최적화: 모바일 완성률 데스크톱 대비 90% 수준
- 시스템 안정성: 99.9% 업타임 달성
