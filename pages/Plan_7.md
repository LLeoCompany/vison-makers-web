1. 프로젝트 목표
   기술적 정체성: RAG(Retrieval-Augmented Generation) 기반 LLM 통합 솔루션.

비즈니스 목표: 기술 비전문가 클라이언트에게 '시간 절감'과 '운영 경험(DevGym)'을 팔아 즉각적인 상담 신청을 유도함.

2. 전체 섹션 구성 (Layout)
   Section 1: Hero (Problem-Solving First)
   비주얼: 좌측(Old: 흑백, 엑셀 120분) vs 우측(New: 컬러, AI 3초) 대비.

카피: "당신의 5시간, 비전메이커에선 5초면 충분합니다."

UX: 스크롤 시 '92% 정확도', '78% 자동화' 수치가 카운팅됨.

Section 2: Solution Matrix (3-Card System)
Card A (주문 자동화): "영업사원 챗 한 줄 -> ERP 입력 완료"

Card B (도면/문서 검토): "수만 장의 규약 대조, 20분 만에 위반 마킹"

Card C (체육관/서비스 추천): "유저 데이터 기반 개인화 매칭 (DevGym 사례)"

Interaction: 카드 클릭 시 해당 테마에 맞는 [원스톱 팝업] 오픈.

Section 3: One-Stop Popup (4-Step Story)
Slide 1 (The Magic): 실제 업무가 3초 만에 처리되는 로그(Log) 애니메이션.

Slide 2 (The Source): AI 답변 옆에 [근거 확인] 버튼. 클릭 시 PDF 원문 하이라이트 노출. (RAG 신뢰 구축)

Slide 3 (The Proof): DevGym 실제 운영 성과 시각화 (그래프/수치).

Slide 4 (The Action): 내장형 상담 폼. "지금 우리 비즈니스 진단받기".

폼 필드: 이름, 연락처, 고민 영역(카드 선택 자동 연동).

Section 4: Technical Authority (Invisible RAG)
비주얼: '디지털 도서관 사원' 애니메이션 (데이터 검색 -> 분석 -> 답변).

카피: "우리는 거짓말하지 않는 AI를 만듭니다. 이것이 비전메이커의 RAG 아키텍처입니다."

3. 이론적 논리 및 기법 (Logical Foundation)
   인지 부하 이론: 전문 용어 대신 '3초', '300% 상승' 등 구체적 수치 위주 정보 전달.

이중 코딩 이론: 모든 텍스트 설명 옆에 해당 동작을 묘사하는 UI 애니메이션 배치.

자이가르닉 효과: 팝업 상단에 진척도 바(Progress Bar)를 배치하여 상담 신청 완수를 유도.

사회적 증거: 직접 운영 중인 DevGym을 실체적 증거로 활용하여 외주사 리스크 제거.

4. UI/UX 구현 디테일
   Color: Background #1A1A2E, Primary #E94560, Success #48BB78.

Motion: Framer Motion 또는 GSAP를 활용한 부드러운 섹션 전환 및 모달 애니메이션.

Mobile: 모달 내에서 모든 플로우가 엄지손가락 하나로 제어 가능하도록 설계.
