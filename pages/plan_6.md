1. 프로젝트 메타 정보
   브랜드명: Vision-Makers (비전메이커)

핵심 키워드: RAG, LLM, 업무 자동화, DevGym 기반 검증

디자인 컨셉: 하이테크 신뢰 (Dark Mode, #1A1A2E / 포인트 #E94560)

타겟: AI 도입을 고민하는 비즈니스 결정권자 (비전문가)

2. 페이지 섹션별 상세 명세
   Section 1: Hero (The High-Speed Contrast)
   목적: 120분 수작업과 3초 AI의 극명한 대비로 뇌에 강력한 자극 부여.

UI 구조: - 2컬럼 레이아웃 (Mobile 시 상하 배치).

좌측(Problem): 흑백/그레이톤. 엑셀 아이콘과 함께 "수작업 120분" 텍스트가 깜빡임.

우측(Solution): 비비드한 컬러. 챗봇 대화창 애니메이션 "주문 처리해줘" -> "3초 만에 완료 ✅".

핵심 카피: - H1: "당신의 5시간, 비전메이커에선 5초면 충분합니다."

Sub: "RAG 기반 맞춤형 AI가 사내 지식 검색부터 ERP 연동까지 스스로 판단하고 처리합니다."

Section 2: Solution Matrix (Problem-Solving Cards)
목적: 기술 이름이 아닌 '내가 겪는 문제'를 선택하게 유도.

UI 구조: 3개의 대형 카드 (Hover 시 심도 효과).

카드 구성:

[주문/ERP 자동화] "영업사원의 한 줄 채팅이 즉시 ERP 주문으로 전송됩니다."

[도면/문서 검토] "수만 장의 규정집과 도면 대조, AI가 20분 만에 위반 사항을 마킹합니다."

[지능형 추천/관리] "유저의 취향과 데이터를 분석해 매출이 발생하는 경로를 제안합니다."

Section 3: Interactive Proof (The Librarian Agent)
목적: RAG를 '도서관 사원' 비유로 설명하여 '환각(거짓말)' 공포 해소.

UI 구조: - 중앙에 '지식 저장소' 아이콘 배치.

질문 입력창 제공 (예시 질문: "우리 회사 복지 규정 알려줘").

핵심 기능: 답변 우측 하단에 [출처 확인] 버튼 생성. 클릭 시 답변의 근거가 된 문서 원본 페이지(PDF 형태)를 오버레이로 노출.

Section 4: Social Proof (DevGym 실전 검증)
목적: "우리는 이론가가 아닌 실전자다"라는 메시지 전달.

UI 구조: 대시보드 스타일의 통계 섹션.

데이터 포인트:

"응답 정확도 92%"

"자동 처리율 78%"

"야간 상담 전환율 300% 상승"

설명: "10만 유저가 사용하는 DevGym 플랫폼을 직접 구축하고 운영하며 쌓은 데이터 노하우를 제공합니다."

3. 기술적 구현 요구사항 (Vibe Coding Prompt)
   Plaintext

1. 모든 수치(%, 초, 분)는 페이지 스크롤 시 0부터 올라가는 '카운팅 애니메이션'을 적용할 것.
1. '출처 확인' 인터랙션은 사용자에게 "이 AI는 팩트에 기반한다"는 확신을 주어야 하므로, 시각적으로 매우 강조할 것.
1. 폼(Form) 제출 버튼은 클릭 시 "비즈니스 진단 중..."이라는 로딩 상태를 보여준 뒤 완료 메시지를 띄울 것.
1. RAG 파이프라인 시각화는 '데이터 수집 -> 인덱싱 -> 검색 -> 생성' 단계를 아이콘 위주로 아주 단순하게 표현할 것.
1. 최종 점검 리스트 (Checklist)
   [ ] 가치 중심인가? RAG라는 단어보다 '시간 절감'이 먼저 보이는가? (Yes)

[ ] 신뢰도가 높은가? DevGym이라는 실제 운영 사례가 포함되었는가? (Yes)

[ ] 직관적인가? 도서관 사원 비유와 출처 배지가 포함되었는가? (Yes)

[ ] 행동을 유도하는가? 무료 진단 리포트라는 명확한 CTA가 있는가? (Yes)
