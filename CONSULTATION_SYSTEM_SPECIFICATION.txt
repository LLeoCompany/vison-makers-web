                              1. 프로젝트 개요
===============================================================================

1.1 프로젝트 배경
현재 LeoFitTech 웹사이트의 상담 신청 시스템은 단순한 폼 구조로 되어 있어:
- 상담 완료율이 상대적으로 낮음
- 고객의 구체적 요구사항 파악 어려움
- 영업팀의 초기 상담 시간 과다 소요

1.2 개선 목표
- 상담 신청 완료율 20% 향상
- 고객 요구사항 정확도 50% 개선
- 영업팀 초기 상담 시간 30% 단축
- 사용자 만족도 4.0/5 이상 달성

1.3 핵심 설계 원칙
1) 단순성 우선: 모든 사용자가 쉽게 완료할 수 있는 구조
2) 일관성 보장: 모든 디바이스에서 동일한 경험 제공
3) 선택 제한: 각 단계 최대 5개 선택지로 부담 경감
4) 일반인 언어: 전문 용어 배제, 직관적 표현 사용
5) 개인정보 최소화: 상담에 필요한 최소한의 정보만 수집

Frontend
- Framework: Next.js 14 (Pages Router 유지)
- Language: TypeScript
- Styling: 기존 SCSS + Tailwind CSS
- 상태 관리: React Context API

Backend
- API: Next.js API Routes (기존 방식 유지)
- 데이터 저장: 기존 Slack 연동 + 이메일 발송
- 임시 저장: Browser LocalStorage

분석 및 모니터링
- Google Analytics 4 (기존 설정 활용)
- 기본적인 A/B 테스트 (Google Optimize)

interface GuidedConsultation {
  // 메타 정보
  id: string;
  type: 'guided';
  submittedAt: Date;
  userAgent: string;
  referrer: string;

  // Step 1: 서비스 종류
  serviceType: 'homepage' | 'shopping' | 'booking' | 'membership' | 'other';

  // Step 2: 규모와 예산
  projectSize: 'small' | 'medium' | 'large';
  budget: '100-300' | '300-800' | '800-1500' | '1500+' | 'consult';

  // Step 3: 일정과 특별 요청
  timeline: '1month' | '2-3months' | '6months' | 'flexible';
  importantFeatures: string[]; // ['mobile', 'seo', 'admin', 'payment']
  additionalRequests?: string;

  // Step 4: 연락처
  contact: {
    name: string;
    phone: string;
    email: string;
    company?: string;
    preferredContactTime: 'morning' | 'afternoon' | 'evening' | 'anytime';
  };
}

===============================================================================
                            5. 기능 요구사항
===============================================================================

5.1 기본 기능
1) 트랙 선택
   - 가이드 트랙 vs 자유 트랙 선택
   - 각 트랙별 특징과 소요시간 안내
   - 선택 도움말 제공

2) 가이드 트랙 (4단계)
   - 단계별 진행상태 표시
   - 이전/다음 버튼으로 네비게이션
   - 각 단계별 입력 검증
   - 진행 중 자동 저장 (LocalStorage)

3) 자유 트랙 (1단계)
   - 자유 텍스트 입력 (최대 2000자)
   - 작성 가이드 제공
   - 실시간 글자수 카운트

4) 데이터 제출
   - Slack 알림 발송
   - 이메일 확인 발송
   - 완료 페이지로 리다이렉트

5.2 부가 기능
1) 임시 저장 및 복원
   - 브라우저 LocalStorage 활용
   - 페이지 새로고침 시 데이터 복원
   - 30일 후 자동 삭제

2) 반응형 디자인
   - 모바일/태블릿/데스크톱 최적화
   - 터치 친화적 인터페이스
   - 모든 디바이스에서 동일한 경험

3) 접근성 지원
   - 키보드 내비게이션 지원
   - 스크린 리더 호환
   - 색상 대비 4.5:1 이상 유지
   - 최소 16px 폰트 크기

5.3 성능 요구사항
- 각 페이지 로딩 시간: 2초 이내
- 페이지 간 전환 시간: 1초 이내
- 99% 업타임 보장
- 모바일 성능 점수 90점 이상

6.1 보안 요구사항
1) 개인정보보호
