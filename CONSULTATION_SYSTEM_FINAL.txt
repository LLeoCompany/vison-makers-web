설계 철학 및 원칙
===============================================================================

핵심 철학
"완벽한 시스템보다는 사용자가 실제로 완료할 수 있는 단순한 시스템"

설계 원칙 (5가지)
1. 단순성 우선: 모든 디바이스에서 동일한 경험
2. 3-5 선택 원칙: 과부하 방지를 위한 선택지 제한
3. 일반인 언어: 전문 용어 배제, 쉬운 설명
4. 최소 정보 수집: 꼭 필요한 정보만 수집
5. 점진적 개선: 완벽함보다 실용성과 완료율 중시

목표 재설정
- 상담 완료율: 기존 대비 20% 향상 (현실적 목표)
- 사용자 만족도: 4.0/5 이상
- 구현 기간: 20주 (현실적 일정)
- 운영 효율성: 초기 상담 시간 30% 단축

시스템 구조 (대폭 단순화)
===============================================================================

2트랙 시스템 (하이브리드 제거)
1. 🎯 가이드 상담: 4단계 간단한 질문
2. 📝 자유 상담: 1페이지 자유 작성

라우팅 구조 (단순화)
/consultation/
├── start              # 트랙 선택
├── guided/            # 가이드 트랙 (4단계)
│   ├── step1         # 서비스 종류
│   ├── step2         # 규모와 예산
│   ├── step3         # 일정과 특별 요청
│   └── step4         # 연락처
├── free/             # 자유 트랙 (1단계)
│   └── write         # 자유 작성 + 연락처
└── thanks            # 완료 페이지

기술 스택 (현실적)
- Frontend: 기존 Next.js Pages Router 유지
- 상태 관리: Context API (기존 방식 유지)
- 저장: LocalStorage + 단순 API 호출
- 분석: Google Analytics (기존 도구 활용)

UI/UX 개선사항
===============================================================================

1. 일관된 디자인 시스템
- 모든 페이지 동일한 레이아웃
- 통일된 버튼, 폼, 색상 체계
- 동일한 진행 표시 방식

2. 명확한 진행 상태
- 가이드: "1/4 단계" 명시적 표시
- 자유: "한 페이지로 완료" 안내
- 완료까지 예상 시간 표시

3. 사용자 친화적 언어
- 전문 용어 제거 ("PWA" → "모바일 앱 같은 웹사이트")
- 구체적 설명 ("5-10 페이지 정도")
- 안심시키는 문구 ("대부분 고객님들이...")

4. 선택 부담 경감
- 각 단계 최대 5개 선택지
- "잘 모르겠어요" 옵션 제공
- 추천 옵션 명시 (✨ 표시)

5. 접근성 보장
- 16px 이상 폰트 크기
- 명확한 색상 대비
- 키보드 네비게이션 지원
- 모든 기기에서 동일한 경험

기술 구현 계획 (현실적)
===============================================================================

Phase 1: 기반 작업 (4주)
- 현재 /sub 페이지 분석 및 백업
- 새로운 라우팅 구조 구축 (/consultation/*)
- 기본 컴포넌트 제작 (Layout, Form, Button 등)
- Context API 상태 관리 구축

Phase 2: 가이드 트랙 구현 (6주)
- Step 1-4 페이지 구현
- 폼 검증 및 진행 상태 관리
- LocalStorage 자동 저장 기능
- 기존 Slack 연동 수정

Phase 3: 자유 트랙 구현 (2주)
- 자유 작성 페이지 구현
- 텍스트 에디터 최적화
- 두 트랙 통합 테스트

Phase 4: 완성 및 테스트 (4주)
- 완료 페이지 및 이메일 알림
- 전체 시스템 테스트
- 성능 최적화
- 접근성 검증

Phase 5: 배포 및 안정화 (4주)
- A/B 테스트 준비 (기존 vs 신규)
- 점진적 트래픽 전환 (20% → 50% → 100%)
- 버그 수정 및 사용자 피드백 반영
- 모니터링 시스템 구축

총 구현 기간: 20주 (5개월)

성능 최적화 전략
===============================================================================

1. 로딩 속도 최적화
- 각 페이지 2초 이내 로딩 목표
- 이미지 최적화 및 지연 로딩
- 불필요한 JavaScript 제거

2. 모바일 최적화
- 터치 친화적 인터페이스 (44px 이상 터치 영역)
- 스와이프 제스처 지원 (선택사항)
- 세로 모드 최적화

3. 오프라인 대응
- LocalStorage 활용 임시 저장
- 네트워크 오류 시 재시도 메커니즘
- 사용자에게 명확한 상태 안내

4. SEO 최적화
- 각 페이지 적절한 title, meta 태그
- 구조화된 데이터 마크업
- 사이트맵 업데이트

운영 및 분석 계획
===============================================================================

기본 분석 (Google Analytics 활용)
- 각 단계별 이탈률 추적
- 완료율 (가이드 vs 자유 비교)
- 디바이스별 사용 패턴
- 유입 경로별 전환율

단순한 A/B 테스트

- 버튼 색상 및 위치 테스트
- 진행 상태 표시 방식 테스트
